Overview

What is Compressify?

Compressify is a Unity Editor tool built by Runtime Labs that automates asset compression and project optimization. Instead of manually adjusting texture import settings, audio compression formats, and mesh parameters one-by-one, Compressify applies the right settings for your target platform in one click.

The PRO version adds a full asset intelligence suite: duplicate detection using MD5 hashing, large asset scanning, build report comparison across releases, and an integrated backup manager — all inside a single, purpose-built editor window.

Without Compressify

  • Manual texture settings per asset
  • No per-platform compression profiles
  • Duplicate assets silently bloat builds
  • No build size trend tracking
  • No automated backup before optimization

With Compressify

  • One-click platform-appropriate compression
  • 5 built-in platform profiles (ETC2, ASTC, DXT5, BC7…)
  • MD5-based duplicate detection and auto-removal
  • Build report diff — compare two build outputs
  • Timestamped backup before every optimization run
PRO feature This documentation covers Compressify PRO (v1.0.0). The Duplicate Finder, Large Asset Scanner, Build Comparator, and Backup Manager are exclusive to the PRO tier.
Installation

Getting the tool into your project

Compressify is distributed as a Unity custom package. Import it through the Package Manager or by dropping the folder directly into your project.

01

Import the package

In Unity, go to Assets → Import Package → Custom Package and select the .unitypackage file, or drag the Runtime Labs/ folder into your Assets/ directory. Requires Unity 2021.3 LTS or newer.

02

Verify the folder structure

Ensure the following path exists in your project. The tool will not load icons if this path is wrong:

Assets/ └── Runtime Labs/ └── Compressify/ └── Editor/ ├── Compressify.cs ← main tool └── Icons/ ├── logo.png ├── texture.png └── ... (see Icon Requirements)
03

Place your icons

Compressify uses PNG icons from the Icons/ folder. Create or download the required icons (listed in the Icon Requirements section) and place them there. The tool functions without icons — they simply won't render.

04

Open the window

In Unity, click Tools → Compressify from the menu bar. The window opens at a minimum size of 520×700 and can be docked anywhere in your editor layout.

Editor-only Compressify is wrapped in #if UNITY_EDITOR and does not include any runtime code. It will never increase your build size.
Quick start

Compress your first project in 60 seconds

The fastest path from open project to optimized project — using the Quick Compress tab.

1

Back up your project first

Go to the PRO Suite → Backup tab and click Create Backup. Compressify modifies Unity asset import settings permanently. A timestamped backup is created in a Backups/ folder next to your project root.

2

Confirm backup & select platform

On the Quick Compress tab, tick the "I have backed up my project" checkbox to unlock compression. Then pick your target platform — Android, iOS, PC, WebGL, or Console.

3

Toggle your asset types

Enable the optimizations you want. Each toggle targets a specific asset category. Textures, Audio, Meshes, and Shader stripping are recommended for all platforms. LOD Settings is optional for non-mobile targets.

4

Click Run Quick Compress

Compressify processes each enabled asset type in sequence. Watch the progress bar and Activity Log as each category completes. A final report shows how much space was recovered.

PROGRESS[3/5] Meshes…
60%Running…
What gets changed? Quick Compress only modifies Unity import settings — it does not delete or move any files. Original source assets are untouched. You can revert any change via Unity's Undo history or by restoring your backup.
Interface

Quick Compress tab

The main tab is organized into four numbered sections that guide you through a safe, repeatable workflow.

Section 1
Confirm Backup

A safety gate. You must tick the checkbox before the Run Quick Compress button becomes active. This is intentional — compression is destructive to import settings and should never be run without a known-good backup to restore from.

The Open Backup Manager → link jumps directly to the Backup sub-tab inside PRO Suite.

Section 2
Target Platform

Choose the deployment target for this optimization pass. Each platform applies different compression formats, audio settings, and mesh parameters. See the full Platform Profiles reference below.

The spec line underneath the buttons updates to show the active profile's key settings at a glance — e.g. "ETC2 textures · Mono Vorbis audio · Medium mesh compression" for Android.

Section 3
Select Optimizations

Six toggles control which asset types are included in the compression pass. Each toggle row shows the asset category name, a brief description, and a pill-style on/off switch. Click anywhere on the row to toggle.

Textures
Sets platform-specific format (ETC2/ASTC/DXT5/BC7), enables mipmaps, caps max size at 1024px (2048px for PC/Console). Applied via Unity's TextureImporter.
Audio
Switches to Vorbis compression at 50% quality for all platforms except Console (which uses PCM lossless). Android and iOS also force mono to halve memory.
Meshes
Sets ModelImporter mesh compression to Medium (High for WebGL). Enables Unity's polygon and vertex optimizers. Applied to all model assets found by the AssetDatabase.
Strip Shaders
Calls Clear() on every ShaderVariantCollection in the project. Removes pre-compiled variants that aren't needed, reducing build shader data. Only affects variance collections, not source shaders.
Sprite Atlases
Enables tight packing and texture compression on every SpriteAtlas asset, reducing atlas texture overhead in 2D projects.
LOD Settings
Disables blend shapes, embedded cameras, and embedded lights on all Model assets — settings commonly left on by default that increase memory without visual benefit.
Section 4
Run Compression + Estimated Impact

The RUN QUICK COMPRESS button triggers a confirmation dialog, then processes every enabled asset type in order. A progress bar and status label show the current step.

The Estimated Impact panel on the right shows project size before and after, plus the estimated savings. These figures update live after each compression run using direct filesystem measurement — no guesswork.

A Safe & Recommended notice confirms that no source files are deleted — only import settings are written.

Interface

Basic tab

The Basic tab exposes each optimization as an individual action — useful when you only need to re-run one specific category without touching the others.

[ TX ]

Compress Textures

Runs only the texture compression pass using the currently selected platform profile. Iterates all t:Texture assets.

[ AU ]

Compress Audio

Re-applies audio compression settings to all t:AudioClip assets. Respects the active platform profile for mono/stereo and quality.

[ MS ]

Optimize Meshes

Runs mesh compression and vertex/polygon optimization on all t:Mesh assets. Uses platform profile to determine compression level.

[ RP ]

Build Size Report

Scans all asset paths and logs the top 10 largest files by size to the Activity Log. No changes are made to any assets.

Platform profile applies here too Individual operations in the Basic tab use the same platform profile selected on the Quick Compress tab. Switch back to Quick Compress to change it before running individual operations.
Interface

Advanced tab

Three additional tools, a danger-zone action, and a build hook for fully automated optimization pipelines.

Asset operations

[ SH ]

Strip Unused Shaders

Clears all ShaderVariantCollections. Run this before a release build to cut shader compilation overhead and build size.

[ AT ]

Compress Sprite Atlases

Finds all t:SpriteAtlas assets and enables tight packing + compressed texture format via serialized properties.

[ LD ]

Apply LOD Settings

Strips embedded cameras, lights, and blend shapes from all Model assets. Particularly useful for imported FBX files from third-party sources.

Remove unused assets

Destructive — no undo "Remove Unused Assets" permanently deletes any asset where AssetDatabase.GetDependencies(path).Length ≤ 1. An asset with no dependents is assumed to be unreferenced. Always create a backup before using this. There is no undo once files are deleted from disk.

Unity's GetDependencies tracks explicit asset references — it does not detect programmatic string-based asset loads such as Resources.Load("path"). Assets loaded this way will appear unreferenced and may be incorrectly deleted. Use this feature only if you have eliminated Resources folder usage from your project.

Build hook

Enable Auto-optimize before every build to register a BuildPlayerHandler that runs the full Quick Compress pass automatically each time you trigger a build. The active optimization toggles and platform profile are used.

// What happens when the hook is enabled:
BuildPlayerWindow.RegisterBuildPlayerHandler(OnBuildPlayerDelegate);

// OnBuildPlayerDelegate runs:
RunQuickCompressSteps(); // applies all active toggles
BuildPipeline.BuildPlayer(opts); // then hands off to Unity

Disable the hook when iterating during development — re-compressing all assets before every test build will significantly slow your build times.

PRO Suite

PRO Suite — five specialist tools

The PRO Suite is a set of five sub-tabs for deeper project intelligence, accessible from the PRO SUITE tab in the main navigation.

01 — Dashboard

Project Overview

Shows total project size before and after any PRO scans, and the amount recovered. Run a full PRO scan to update all figures at once.

02 — Duplicates

Duplicate Finder

MD5-hashes every file in Assets/ above a configurable size threshold, groups identical files, and reports how much space duplicates waste.

03 — Large Assets

Large Asset Scanner

Lists all assets exceeding the size threshold (shared with Duplicates), sorted largest-first. Useful for identifying targets before manual optimization.

04 — Build Compare

Build Report Diff

Compares two Unity build reports (text or JSON) line-by-line, showing added and removed assets and the overall size delta between builds.

05 — Backup

Backup Manager

Creates a full recursive copy of your Assets/ folder into ../Backups/Backup_YYYYMMDD_HHMMSS/. Use before any destructive operation.

Using the Duplicate Finder

Set the minimum file size threshold in the Duplicates tab (default: 1 MB). This filters out small files like .meta entries that commonly share content. Click Scan for Duplicates — Compressify reads every qualifying file, computes its MD5 hash, and groups files that share a hash.

Results show the number of groups, total duplicate count, and recoverable space. Expand the Show duplicate groups foldout to inspect paths. Use Copy MD5 Report to export the full list to clipboard.

Auto-Delete Duplicates keeps the first file in each group (alphabetically by path) and deletes the rest using AssetDatabase.DeleteAsset. A confirmation dialog is always shown first.

Review before deleting Files can share the same MD5 hash and serve different logical roles (e.g. two identical textures used in separate materials). Review the group list and verify which copy you want to keep before running auto-delete.

Using the Build Report Comparator

After making two separate builds, copy the generated *.txt or *.json build log files somewhere accessible. Click the … browser buttons to locate Report A (older build) and Report B (newer build), then click Compare Reports →.

The comparator parses line entries, extracts filenames and sizes, and produces a diff showing assets added in B, assets removed compared to A, and the total size delta. Results appear in the output area below and can be scrolled.

The Backup Manager

Click Create Backup to copy your entire Assets/ folder to <ProjectRoot>/Backups/Backup_YYYYMMDD_HHMMSS/. The operation is synchronous and may take a few seconds on large projects. A success message shows the full destination path.

Click Open Folder to reveal the Backups directory in your OS file manager. Old backups can be manually deleted from there — Compressify does not manage backup cleanup automatically.

Backup location Backups are stored outside your Assets/ folder so they are not imported by Unity and do not affect your project. The path is <ProjectRoot>/Backups/.
Reference

Platform profiles

Each profile encodes the best-practice compression settings for that deployment target. All settings are applied through Unity's standard TextureImporter, AudioImporter, and ModelImporter APIs — they are exactly what you would set manually in the Inspector.

Profile Texture format Max texture size Audio Mesh compression
Android ETC2_RGBA8 1024 px Vorbis 50%, Mono Medium
iOS ASTC_6x6 1024 px Vorbis 50%, Mono Medium
PC DXT5 2048 px Vorbis 50%, Stereo Medium
WebGL DXT5 1024 px Vorbis 50%, Stereo High
Console BC7 2048 px PCM lossless, Stereo Medium

Why these formats?

ETC2 is the mandatory hardware-decoded format on all Android 4.3+ devices and has universal OpenGL ES 3 support. It gives excellent ratio with no quality loss on devices.

ASTC 6×6 is the flexible successor for iOS 8+ and modern Android. The 6×6 block size balances size reduction against quality — use ASTC 4×4 for higher-quality UI textures if needed.

DXT5 (BC3) is the universal choice for DirectX targets. WebGL also benefits since the browser decompresses DXT on the GPU for most desktop users.

BC7 is the highest-quality DX11 block-compressed format, appropriate for the higher-spec hardware of console and high-end PC targets where the texture budget is less critical.

Mono audio on mobile halves audio memory footprint with no perceptible difference on device speakers or earbuds. Console and PC use stereo because spatial audio quality matters on those platforms.

High mesh compression on WebGL is set because WebGL downloads are bandwidth-constrained. The tradeoff is slightly longer load time as the GPU decompresses on first use.

FAQ

Frequently asked questions

Will Compressify affect my game's visual quality?
Compression always involves a tradeoff. ETC2, ASTC, and DXT5 are lossy — you may see minor artifacts on textures with fine gradients or text. The default max texture size of 1024px (2048px on PC/Console) may downsample very large textures. For UI-critical textures, consider manually overriding those specific assets after the bulk compression pass.
Can I run Compressify on a project that's already been compressed once?
Yes. Running it again will re-apply the same settings, which is a no-op if nothing changed. If you switch platform profile between runs, the new settings overwrite the previous ones. Import settings are stored in .meta files and are safe to overwrite repeatedly.
How long does Quick Compress take?
It depends on your project size. The bottleneck is Unity's SaveAndReimport() call — each texture, audio clip, and mesh must be reimported after its settings change. A project with 500 textures, 200 audio clips, and 150 meshes typically takes 5–15 minutes. Subsequent runs are faster because Unity caches reimport results.
The Duplicate Finder is slow on my project. Why?
The finder reads every file's bytes to compute its MD5 hash. On a project with thousands of large binary assets (textures, audio, prefabs), this involves significant disk I/O. The 1 MB minimum threshold helps — increase it further to skip smaller files you don't care about. Run the scan during a break or overnight.
Can I undo compression after running it?
Import setting changes appear in Unity's Undo stack for the current session. After restarting Unity, the only way to revert is to restore a backup. This is why the backup confirmation gate exists on the Quick Compress tab — use it every time.
What happens if I compress for Android but then switch to building for PC?
The per-platform override settings applied by Compressify only activate when Unity builds for that specific platform. When you switch build target to PC, Unity will use the PC override settings (or the default settings if PC hasn't been run yet). Platform-specific overrides coexist — they do not conflict.
Is it safe to enable "Auto-optimize before every build"?
Safe, but slow. Every build will trigger a full compression pass before handing off to Unity's build pipeline. This is appropriate for CI/CD pipelines and final release builds, but will significantly slow iterative development builds. Turn it off during day-to-day work and enable it only for release pipeline runs.
Does the tool support Unity 6 / URP / HDRP?
Compressify operates at the asset importer level, which is render-pipeline-agnostic. It works with Built-in RP, URP, HDRP, and Unity 6. The shader stripping feature clears ShaderVariantCollection assets regardless of pipeline. No render-pipeline-specific features are used.
